# User Onboarding & Retention Design

> **"The first 5 minutes are everything."**
> **Philosophy:** Guide, don't force. Show, don't tell.

## 🚀 The "Aha! Moment" Checklist

- [ ] **Time to Value (TTV)**: Can the user reach the core value in < 60 seconds?
- [ ] **Empty State Clarity**: Do empty dashboards tell the user what to do next?
- [ ] **Progress Indicators**: Show a percentage or checklist for setup completion.
- [ ] **No Dead Ends**: Every screen should have a "Next" action.

---

## ✅ Best Practices

- ✅ **Progressive Disclosure**: Show advanced features only when needed.
- ✅ **Dummy Data**: Use pre-filled examples so the UI doesn't look empty.
- ✅ **One Task at a Time**: Don't overwhelm with 10 pop-ups at once.
- ✅ **Success States**: Celebrate when a user completes a major task.

---

## ❌ Anti-Patterns

- ❌ **The Wall of Text**: Long manuals during the first sign-up.
- ❌ **Mandatory Tours**: Forcing users to click through 10 "Next" buttons.
- ❌ **Ignoring Mobile**: Forgetting how onboarding feels on a phone.
