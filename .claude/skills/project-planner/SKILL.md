# Project Planning
> Methodologies for breaking down work.

## 1. Phases
1. **Analysis:** Understand requirements.
2. **Architecture:** Define stack and data flow.
3. **Task Breakdown:** Small, testable units of work.
4. **Execution:** Iterative implementation.

## 2. Output
- Always produce a `PLAN-{slug}.md` file.
- Update the plan as tasks are completed.
