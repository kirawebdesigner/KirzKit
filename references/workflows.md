# Autonomous Workflows Reference

Workflows are structured sequences for high-velocity SaaS building.

## Core Workflows

### /plan - Project Planning Mode
1. **Analyze User Request**: Identify core requirements.
2. **Phase -1: Context Check**: Map current codebase or project state.
3. **Phase 0: Socratic Gate**: Ask clarifying questions before planning.
4. **Generate Plan**: Create `docs/PLAN-{slug}.md` with task breakdown.
5. **Assign Agents**: Map specialists to each phase.

### /ui-audit - Professional Design Critique
1. **Initial Review**: High-level heuristic evaluation.
2. **Pattern Analysis**: Review interaction and layout consistency.
3. **Visual Critique**: Analyze typography, color, and spacing.
4. **Report**: Generate remediation list and priority items.

### /landing-page-teardown - Conversion Analysis
1. **Value Proposition Audit**: Analysis of the primary hook.
2. **Conversion Flow Mapping**: Tracing the user's path to action.
3. **Trust Signals Analysis**: Review of social proof and credibility.
4. **Copywriting Critique**: Assessing the clarity and impact of copy.
5. **Final Strategy**: Providing concrete landing page improvements.

### /refine - Prompt Optimization
1. **Initial Review**: Analyze the input prompt.
2. **Meta-Prompting**: Apply advanced prompt engineering patterns.
3. **Test Simulation**: Predict agent output based on refined prompt.
4. **Final Refinement**: Provide high-signal instruction.

## Additional Workflows
- **accessibility-review**: Ensure WCAG compliance.
- **brainstorm**: Idea generation for features or marketing.
- **debug**: Systematic root-cause analysis and fixing.
- **deploy**: Structured deployment checklist and execution.
- **test**: Comprehensive testing strategy and coverage.
- **ui-ux-pro-max**: Full-spectrum design-to-code workflow.
